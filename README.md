# Excalidraw Helm Chart

This Helm chart deploys the Excalidraw frontend (served by Nginx) and backend (Node.js) on a Kubernetes cluster such as **k3s**.

## Prerequisites
- A running Kubernetes cluster (k3s, minikube, kind, etc.)
- `helm` CLI installed (`helm version >= v3`)
- Docker images for the frontend and backend pushed to a registry that your cluster can pull from.  The default `values.yaml` points to the `latest` tag of the images `excalidraw-frontend` and `excalidraw-backend`.

## Installing the chart
```bash
# From the repository root
cd /opt/excalidraw/helm
helm dependency update   # (no dependencies now, but good practice)
helm install excalidraw .
```

## Customising the deployment
Edit `values.yaml` before installing, or supply overrides on the command line:
```bash
helm install excalidraw . \
  --set frontend.image.repository=myregistry.com/excalidraw-frontend \
  --set backend.image.repository=myregistry.com/excalidraw-backend
```

You can also change the service type (e.g., `LoadBalancer` or `NodePort`) and ports.

## Verifying the deployment
```bash
kubectl get pods -l app.kubernetes.io/name=excalidraw-frontend
kubectl get svc -l app.kubernetes.io/name=excalidraw-frontend
kubectl get pods -l app.kubernetes.io/name=excalidraw-backend
kubectl get svc -l app.kubernetes.io/name=excalidraw-backend
```

## Uninstalling
```bash
helm uninstall excalidraw
```
